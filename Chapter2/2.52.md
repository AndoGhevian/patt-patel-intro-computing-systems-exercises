### 2.52 Consider two hexadecimal numbers: x434F4D50 and x55544552. What values do they represent for each of the five data types shown?
1. Unsigned binary
1. 1’s complement
1. 2’s complement
1. IEEE 754 floating point
1. ASCII string

#### Solution

x434F4D50 | x55544552

So we need to reverse, from data type representation to
actual value they express.

#### Unsigned binary

with unsigned integer it is simple, we can consider these both
as binary numbers in positional form.

x434F4D50 = 0 * 16^0 + 5 * 16^1 + 13 * 16^2 +
4 * 16^3 + 15 * 16^4 + 4 * 16^5 + 3 * 16^6 + 4 * 16^7 =

= 1129270608 (I usually use calculator)

x55544552 = 2 + 5 * 16 + 5 * 256 + 4 * 16^3 +  4 * 16^4 +
5 * 16^5 + 5 * 16^6 + 5 * 16^7 =

1431586130

#### 1’s complement

Both are starting with 0 so both represent positive number.
As we know in 1's complement form positive numbers simply
positional binary numbers, so values they represent even in
1's complement are the same.

#### 2’s complement

They again represent same values as in unsigned integer form.

#### IEEE 754 floating point

x434F4D50 = 0100 0011 0100 1111 0100 1101 0101 0000

Now lets distinguish format:

0 1000 0110 1001 1110 1001 1010 1010 00

so we have exponent = 1000 0110 - 0111 1111 = 0000 0111 = 7

1.1001 1110 1001 1010 1010 00 * 2^7 =

= 11001111.0100110101010 =

= 128 + 64 + 8 + 4 + 2 + 1 + 0.25 + 0.03125 + 0.015625
+ 0.00390625 + 0.0009765625 + 0.000244140625 =

= 207.302002

x55544552 = 0101 0101 0101 0100 0100 0101 0101 0010 =

0 1010 1010 1010 1000 1000 1010 1010 010

Ok now, here is idea, we can append in the end as many 0's as we wish,
and so we can express this whole thing in hex format, will be easier to calculate subsequently.

exponent = 1010 1010 - 0111 1111 = 0010 1011 = 32 + 8 + 2 + 1 =
43

1.1010 1000 1000 1010 1010 010 * 2^43 =

11010 1000 1000 1010 1010 010 * 2^20

1101 0100 0100 0101 0101 0010 * 2^20

xD44552 * 16^5 = (2 + 5 * 16 + 5 * 16^2 + 4 * 16^3 + 4 * 16^4
+ 13 * 16^5) * 16^5 = 13911378 * 16^5
