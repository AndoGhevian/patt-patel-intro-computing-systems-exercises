### 2.39 Write IEEE floating point representation of the following decimal numbers.
1. a. 3.75
1. b. −55 | 23/64
1. c. 3.1415927
1. d. 64,000 (integer)

#### Solution

What about IEEE floating point data type representation
using 32 bit representation.

exponent - 127 = x =>

exponent = x + 127 = x + 1...1 (7 bits), where exponent is not
00..00 and is not 11..11 (all 1's and all 0's)

So we have negative exponentes are representd by

0...01 = -126 through 011...10 = -1

and non-negative exponents are represented by

011...11 = 0 through 11...10 = 127

#### a. 3.75

3 = 11

0.75 = 2^-1*a-1 + 2^-2*a-2 + ....

a-1 = 1

0.5 = 2^-1*a-2 + 2^-2*a-3 + ....

a-2 = 1

0 = 2^-1*a-3 + ...

rest of coefitients are 0.

3.75 = 11.11 = 1.111 * 2^1

1(sign) 10000000 (8 bit excess coded exponent)
11100000000000000000000 (23 bits fraction part),

23 bits fraction part and implicit 1 as a normalized scientific
notation leading non zero digit.

#### b. −55 | 23/64

-55 = -(32 + 16 + 4 + 2 + 1) = -110111

23/64 = 2^-1*a1 + 2^-2*a2 + ...

46/64 = a1 + 2^-1*a2 + 2^-2*a3 + ...

a1 = 0

92/64 = a2 + 2^-1*a3 + ...

a2 = 1

56/64 = a3 + 2^-1*a4 + ...

a3 = 0

112/64 = a4 + 2^-1*a5 + ...

a4 = 1

48/32 = a5 + 2^-1*a6 + ...

a5 = 1

32/32 = a6 + 2^-1*a7 + ...

a6 = 1

So 23/64 = 0.010111

-55 | 23/64 = -110111.010111 = -1.10111010111 * 2^5

1 (sign bit) 10000100 (8 bit exponent excess code for 5)

10111010111000000000000 (23 fraction bits / 24 bits precision including 1 left to binary point)
